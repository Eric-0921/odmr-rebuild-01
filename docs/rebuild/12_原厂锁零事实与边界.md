# 原厂锁零事实与边界

这份文档只服务一个目的：

**给后续 agent 一个不会误读的原厂锁零 source of truth。**

它不描述 GUI，不描述当前 runtime 细节，不描述物理磁场真值验证，只回答：

- 原厂“锁定零场”在代码里到底是什么意思
- 原厂真实做了什么
- 原厂没有做什么
- `odmr-rebuild-01` 当前在这个基础上额外加了什么

## 2. 先钉死的结论

### 2.1 原厂 `LockZero` 不是物理零磁场闭环

原厂 `LockZero` 的真实语义是：

**锁定零偏电流基线，然后在这个基线上叠加复现电流。**

它不是：

- 先测出物理磁场真的等于 0
- 再把物理零磁场闭环锁住

### 2.2 原厂输出模型只有一条核心公式

```text
total_current = zero_offset_current + recurrence_current
```

对应原厂变量：

- `zero_offset_current` = `zeroSetCurr`
- `recurrence_current` = `recurSetCurr`

### 2.3 原厂回读的是电流，不是磁场真值

原厂依赖的是 `MEAS:CURR?` 电源回读。

在锁零并输出开启时，它按：

```text
recur_current = measured_total_current - zeroSetCurr
recur_field = recur_current * coilConstant
```

恢复“复现电流 / 复现磁场”的显示值。

### 2.4 所以原厂最多支持这句话

```text
原厂实现了零偏电流锁定，并在锁零状态下输出 零偏 + 复现 电流。
```

原厂**不支持**下面这句话：

```text
原厂已经验证了物理零磁场。
```

## 3. 原厂状态变量

按 `FormMain.cs`，每个轴本质上都有这些量：

| 变量 | 含义 | 单位 | 来源 |
|---|---|---|---|
| `zeroSetCurr` | 零偏电流设定值 | mA | UI 输入，持久化到 `para.xml` |
| `recurSetCurr` | 复现电流设定值 | mA | UI 输入/计算 |
| `recurSetMag` | 复现磁场设定值 | nT | UI 输入/计算 |
| `coilConstant` | 线圈常数 | nT/mA | `para.xml` |
| `LockZero` | 锁零开关 | bool | UI toggle |
| `Output` | 输出开关 | bool | UI toggle |
| `totalCurr` | 实际发给电源的总输出电流 | mA | 运行时计算 |
| `MEAS:CURR?` 回读 | 电源实际输出电流 | A | 设备回读 |

## 4. 原厂配置是怎么保存的

原厂把零偏和线圈常数都保存在本地 XML：

- `ReadCoilConstant / SaveCoilConstant`
- `ReadZeroOffset / SaveZeroOffset`

对应代码在 [LocalSettingAccessor.cs](../../reverse_application/reverse_output/decompiled/SimplePowerController/SimplePowerController/LocalSettingAccessor.cs)。

这说明原厂的“零”首先是一个**持久化电流参数**，不是一个由外部磁场计实时修正的物理量。

## 5. 原厂的两种工作模式

### 5.1 `LockZero = OFF`

行为：

- 输出电流 = `zeroSetCurr`
- `recurSetCurr` / `recurSetMag` 只参与显示或换算
- 不参与实际输出

证据：

- [逆向分析报告-协议与算法还原.md](../../reverse_application/reverse_output/逆向分析报告-协议与算法还原.md) 的 3.2 节
- [FormMain.cs](../../reverse_application/reverse_output/decompiled/SimplePowerController/SimplePowerController/FormMain.cs) 中 `cToggleButtonOutputX/Y/Z_CheckedChanged`

### 5.2 `LockZero = ON` 且 `Output = ON`

行为：

- 输出电流 = `zeroSetCurr + recurSetCurr`
- 改 `recurSetCurr`，会重算总电流并重新发 `CURR`
- 改 `recurSetMag`，会先按 `recurSetCurr = recurSetMag / coilConstant` 换算，再发 `CURR`
- 改 `zeroSetCurr`，也会重算总电流

证据：

- [逆向分析报告-协议与算法还原.md](../../reverse_application/reverse_output/逆向分析报告-协议与算法还原.md) 的 3.2 节
- [FormMain.cs](../../reverse_application/reverse_output/decompiled/SimplePowerController/SimplePowerController/FormMain.cs) 中 `cToggleButtonLockZeroX/Y/Z_CheckedChanged`

## 6. 输出开关到底做了什么

### 6.1 打开输出

当某一轴 `Output` 从 OFF 变成 ON：

- 若 `LockZero = OFF`，发 `zeroSetCurr`
- 若 `LockZero = ON`，发 `zeroSetCurr + recurSetCurr`
- 然后发送 `OUTP 1`
- 再把该轴 `LockZero` toggle 置为可用

证据在 [FormMain.cs](../../reverse_application/reverse_output/decompiled/SimplePowerController/SimplePowerController/FormMain.cs) 的 `cToggleButtonOutputX/Y/Z_CheckedChanged`。

### 6.2 关闭输出

关闭输出时，原厂代码并不完全对称：

- X 轴：先 `CURR 0`，再 `OUTP 0`
- Y/Z 轴：直接 `OUTP 0`

这说明原厂自己也不是一个“完全严格一致的硬件状态机”实现。后续 agent 不应把原厂行为写得比它实际更整洁。

## 7. 锁零开关到底做了什么

### 7.1 打开锁零

当 `LockZero` 从 OFF 变成 ON：

- 立刻把输出改成 `zeroSetCurr + recurSetCurr`
- 禁止继续编辑 `zeroSetCurr`

### 7.2 关闭锁零

当 `LockZero` 从 ON 变成 OFF：

- 立刻把输出改回 `zeroSetCurr`
- 重新允许编辑 `zeroSetCurr`

证据都在 [FormMain.cs](../../reverse_application/reverse_output/decompiled/SimplePowerController/SimplePowerController/FormMain.cs) 的 `cToggleButtonLockZeroX/Y/Z_CheckedChanged`。

## 8. 原厂怎么“测零”

原厂没有看到“外部磁场传感器测到 0 再锁住”的逻辑。

它实际做的是：

1. 输出打开
2. 定时轮询 `MEAS:CURR?`
3. 在 `LockZero = OFF` 时，把总电流回读显示为零偏回读
4. 用户把此时偏置当作零基线
5. 再打开 `LockZero`

回读处理在 [FormMain.cs](../../reverse_application/reverse_output/decompiled/SimplePowerController/SimplePowerController/FormMain.cs) 的 `RecurrenceX/Y/Z_DataReceived`。

## 9. 回读显示的真实语义

### 9.1 锁零且输出开启

原厂按下式恢复“复现”：

```text
recur_current = measured_total_current - zeroSetCurr
recur_field = recur_current * coilConstant
```

### 9.2 没有锁零

原厂直接把总电流回读显示成“零偏回读值”。

所以原厂所谓“零偏回读”，本质上只是当前模式下的电流显示语义，而不是独立传感器给出的“真实零磁场测量”。

## 10. 原厂做了哪些物理假设

### 10.1 磁场和电流线性对应

原厂采用：

```text
field_nt = current_ma * coilConstant_nt_per_ma
current_ma = field_nt / coilConstant_nt_per_ma
```

证据：

- [逆向分析报告-协议与算法还原.md](../../reverse_application/reverse_output/逆向分析报告-协议与算法还原.md) 的 3.1 节
- [LocalSettingAccessor.cs](../../reverse_application/reverse_output/decompiled/SimplePowerController/SimplePowerController/LocalSettingAccessor.cs) 的 `ReadCoilConstant`

### 10.2 输出电流按正值发送

原厂真正发命令时用了：

```text
CURR {Math.Abs(totalCurr) / 1000.0:f5}
```

所以不能默认认为原厂已经实现了双向正负电流控制。

## 11. 原厂明确没有做的事

下面这些能力，当前没有在原厂代码里看到直接证据：

- 用独立磁场传感器测量物理磁场
- 根据物理磁场误差自动修正 `zeroSetCurr`
- 三轴耦合补偿
- 物理零磁场闭环验证
- 基于磁场真值的自动校准

## 12. `odmr-rebuild-01` 在这个基础上额外加了什么

这部分不是原厂行为，是 rebuild 当前新增的工程化链路。

### 12.1 rebuild 新增了 `target_b_nt -> calibration -> target_current_a`

也就是说，rebuild 会先把目标磁场通过 calibration 换算成目标电流。

### 12.2 rebuild 新增了 `measured_current_a` 回读闭环

当前 `hardware verify-mag-lock` 和 `run execute` 在零偏锁定后，会继续验证：

```text
target_b_nt
  -> calibration
  -> target_current_a
  -> measured_current_a
```

这条链是 **rebuild 的工程化验证链**，不是原厂完整照搬出来的“最终物理真值链”。

### 12.3 rebuild 仍然只验证到电流层

当前 rebuild 最准确的话术仍然是：

- 已验证零偏电流锁定
- 已验证 `target_b_nt -> calibration -> target_current_a -> measured_current_a`
- 未验证物理零磁场已经被独立证明

## 13. 给后续 Agent 的固定话术

### 可以这样写

- 原厂“锁定零场”的真实实现是零偏电流基线锁定。
- 原厂输出模型是 `总电流 = 零偏电流 + 复现电流`。
- 原厂的“复现磁场”是由电流回读结合线圈常数换算出来的显示值。
- `odmr-rebuild-01` 在此基础上又加了 `target_b_nt -> calibration -> target_current_a -> measured_current_a` 的验证链。

### 不要这样写

- 原厂已经证明磁场被锁到 0。
- 原厂有真实磁场闭环。
- 原厂零场锁定等于物理零磁场锁定。
- rebuild 已经证明 3D 磁场物理真值成立。
