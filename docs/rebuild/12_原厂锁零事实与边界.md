# 原厂锁零事实与边界

本文只记录从 `reverse_application/reverse_output` 反编译结果中能直接确认的事实，不夹带 rebuild 自己的推断扩写。

## 1. 原厂“锁零”锁的是零偏电流，不是物理零磁场

从 `SimplePowerController` 反编译代码能确认：

- 原厂保存的是三个轴的 `ZeroOffset`
- `ZeroOffset` 只是本地配置里的零偏电流值
- 没有看到任何独立磁场传感器参与闭环

这意味着原厂所谓“锁定零场”，本质上是：

- 先确定零偏电流基线
- 再把复现电流叠加到这个基线上

不是“已经测到物理磁场为 0 并持续闭环修正”。

## 2. 原厂输出模型是“零偏电流 + 复现电流”

打开输出或切换锁零状态时，原厂的核心语义是：

- `LockZero = OFF` 时，输出电流是 `zeroSetCurr`
- `LockZero = ON` 时，输出电流是 `zeroSetCurr + recurSetCurr`

所以原厂控制语义不是“目标磁场直接闭环”，而是：

- 零偏电流基线
- 加上按线圈常数换算出的复现电流

## 3. 原厂验证手段只有电流回读

反编译代码里能确认的回读只有：

- `MEAS:CURR?`

并且 UI 的回读解释也是电流语义：

- 锁零开着时，用 `measured_total_current - zeroSetCurr` 作为复现电流
- 再乘线圈常数显示为复现磁场

这里仍然没有出现“外部磁场传感器测得真实磁场后再修正”的链路。

## 4. 原厂磁场换算只是线圈常数乘除

反编译结果和逆向报告都表明：

- `mag -> curr = mag / coilConstant`
- `curr -> mag = curr * coilConstant`

这说明原厂“磁场”显示是基于线圈常数的换算值，不是独立测量值。

## 5. 还有一个边界：原厂发送电流时取绝对值

反编译代码里存在：

- `CURR {Math.Abs(totalCurr) / 1000.0:f5}`

这进一步说明原厂实现并不是一个完整的正负电流双向物理闭环。

## 6. 对 rebuild 文案的约束

因此 rebuild 文档里允许写：

- `baseline_current_a`
- `locked_zero_offset_current_a`
- `target_b_nt -> calibration -> target_current_a -> measured_current_a`
- “零偏电流锁定”
- “电流回读一致性验证”

但不允许写成：

- “物理零磁场已证明”
- “磁场闭环已验证”
- “已经用外部磁场测量证明锁零成功”

## 7. 当前 rebuild 该怎么表述

当前最准确的表述是：

- 原厂旧系统提供的是“零偏电流基线 + 复现电流”的控制语义
- rebuild 当前验证的是 `target_b_nt -> calibration -> target_current_a -> measured_current_a` 的电流链
- 这条链说明：
  - 目标磁场设定能通过 calibration 翻译到目标电流
  - Maynuo 的输出与 `MEAS:CURR?` 回读在 run 中可核对
- 但这仍然不是物理磁场闭环验证
