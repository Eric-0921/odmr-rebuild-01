# 原标签契约

本文只定义当前 runtime 已经落盘、可直接给后续离线流程消费的“原标签”。不定义峰位、峰宽、对比度、SNR、机器学习训练标签。

## 一、最终事实来源

真正不可替代的事实层有三类：

- `raw/oe1022d.rall`
- `raw/oe1022d.frames.idx.jsonl`
- `segments.jsonl`

当前在 debug 模式下还可同步落一层逐帧结构化 companion truth：

- `raw/oe1022d.frames.parsed.jsonl`

含义：

- `raw/oe1022d.rall` 是连续原始 `RALL?` 数据流
- `raw/oe1022d.frames.idx.jsonl` 是每帧的时间戳、monotonic_ns、raw_offset、raw_len、frame_seq 真值
- `segments.jsonl` 把连续流中的 point 边界归属到 point

所以后续如果要重做解析、重做切分、重做离线标签，必须以这三类文件为准。

`raw/oe1022d.frames.parsed.jsonl` 的用途是：

- 避免每次都从 raw 裸解析 20 x 50 表格
- 让 point 聚合和离线 replay 直接复用同一份逐帧结构化结果
- 作为调试/核对重型 sidecar，而不是默认运行时主产物

但它不是替代 raw 的主真值。

## 二、当前 point 原标签

### 1. `points.jsonl`

提供：

- `point_id`
- `target_b_nt`
- `baseline_current_a`
- `calibrated_delta_current_a`
- `target_current_a`
- 当前 point 使用的 RF sweep 参数
- settle/readback 结果

这层回答的是：

- 这个 point 想测什么
- runtime 实际想把磁场和 RF 设置到什么状态

### 2. `point_fields.jsonl`

提供：

- `segment_id`
- `frames_parsed`
- `samples_total`
- `matrix_shape`
- `measurement_field_order`
- `measurement_field_keys`
- `field_summaries`
- `b_pll_locked_ratio`
- `b_input_overload_ratio`
- `b_gain_overload_ratio`
- 最后一个参考状态摘要
- `sidecar.relative_path`

这层回答的是：

- 这个 point 对应的数据窗口里，完整字段数据放在哪、摘要统计是什么

完整 20 字段数组与必要状态数组默认保存在：

- `point_fields/<segment_id>.npz`

### 3. `quality.jsonl`

提供：

- `frames_total`
- `frames_unique`
- `duplicate_count`
- `duplicate_ratio`
- `timeout_count`
- `last_frame_age_ms`
- `quality_status`

这层回答的是：

- 这个 point 的数据质量能不能被当成有效样本

## 三、当前不属于 runtime 原标签的内容

这些内容明确后置：

- 峰位
- 峰宽
- 对比度
- SNR
- 谱峰拟合结果
- ODMR 物理解释标签
- 机器学习训练标签

原因不是这些不重要，而是它们都不该进入当前采集线程。

## 四、给后续离线处理的最小拼装方式

如果后续要做离线样本构建，当前应该按下面的关系拼：

```text
point_id
  -> points.jsonl         (实验上下文)
  -> segments.jsonl       (point 边界真值)
  -> point_fields.jsonl   (轻量 metadata)
  -> point_fields/*.npz   (完整 20 字段数组 + 必要状态数组)
  -> quality.jsonl        (质量状态)
  -> raw/oe1022d.rall     (最终原始事实)
  -> raw/oe1022d.frames.idx.jsonl (帧级时间戳和 raw offset 真值)
```

## 五、当前边界为什么是合理的

当前边界的目的是：

- 不污染 collector 热路径
- 不把分析逻辑塞进 runtime
- 保留以后重切 segment、重做解析、重做标签的自由度

所以当前策略是：

- 采集阶段只保证原始 truth、point 归属、最小字段级原标签
- 科学标签和机器学习标签全部后置
