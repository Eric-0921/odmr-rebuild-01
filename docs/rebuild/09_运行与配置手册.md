# 运行与配置手册

本文只讲当前 `odmr-rebuild-01` 已实现的 Windows C# CLI 主链，不讲 GUI，不讲峰拟合，不讲机器学习后处理。旧 Rust CLI 已归档，不再作为日常实验入口。

## 一、当前命令入口

### 1. build

```powershell
dotnet build tools/win-csharp/Odmr.Win.sln
```

用途：

- 编译当前 C# 主栈
- 不需要 Rust toolchain

### 2. device probes

```powershell
dotnet run --project tools/win-csharp/Odmr.WinProbe -- visa-list
dotnet run --project tools/win-csharp/Odmr.WinProbe -- oe-idn --resource ASRL8::INSTR --baud 921600
dotnet run --project tools/win-csharp/Odmr.WinProbe -- smb-probe --host 169.254.2.20 --port 5025
dotnet run --project tools/win-csharp/Odmr.WinProbe -- m8812-probe --x COM4 --y COM6 --z COM3
dotnet run --project tools/win-csharp/Odmr.WinProbe -- laser-probe --port COM9 --off-only
```

用途：

- 验证 VISA / TCP / Serial 连接事实
- 验证设备身份、基本读写和安全 cleanup
- Laser 只做 off-only echo，不开启输出

### 3. run resolve

```powershell
dotnet run --project tools/win-csharp/Odmr.WinProbe -- run-resolve --station configs/stations/lab_a.json --calibration configs/calibrations/main.json --plan configs/plans/minimal_3point_runtime.json --smb-profile configs/profiles/smb100a_run_monitor_2830_2890_-10dbm.json --oe-profile configs/profiles/oe1022d_run_ch_b_observed.json --laser-profile configs/profiles/cni_laser_run_off_background.json
```

用途：

- 不碰设备
- 解析 station / calibration / plan / profiles
- 展开 explicit、raster、bounce_1d_x points
- 预检查 JSON 配置是否能驱动 runtime

### 4. run execute

```powershell
dotnet run --project tools/win-csharp/Odmr.WinProbe -- run-execute --station configs/stations/lab_a.json --calibration configs/calibrations/main.json --plan configs/plans/minimal_3point_runtime.json --smb-profile configs/profiles/smb100a_run_monitor_2830_2890_-10dbm.json --oe-profile configs/profiles/oe1022d_run_ch_b_observed.json --laser-profile configs/profiles/cni_laser_run_off_background.json --out-dir runs/win_csharp_manual_minimal
```

用途：

- 加载 station / calibration / plan / profiles
- 下发 SMB/OE/Laser 固定 profile
- 执行一次零偏电流锁定
- 启动 run 级 `RALL?` collector
- 按 point 执行 sweep-driven 采集，并写出完整 artifact

### 5. offline review

```powershell
dotnet run --project tools/win-csharp/Odmr.WinProbe -- artifact-check --run runs/win_csharp_manual_minimal
dotnet run --project tools/win-csharp/Odmr.WinProbe -- audit-continuity --run runs/win_csharp_manual_minimal --out runs/win_csharp_manual_minimal/continuity_audit.json
dotnet run --project tools/win-csharp/Odmr.WinProbe -- device-command-check
```

用途：

- `artifact-check` 只读 run 目录，不碰设备
- `audit-continuity` 只读 `raw + frames.idx`，不进入 collector
- `device-command-check` 展示 C# 已迁移命令面和 `win-csharp-rebuild` 归档分支来源

## 二、配置文件分别管什么

### 1. station

文件：

- `configs/stations/lab_a.json`

作用：

- 保存设备列表
- 保存 TCP 地址或最近成功端口作为 hint
- 保存设备 identity 条件
- SMB100A 可用 `host_candidates` 配多个固定地址，按顺序尝试
- OE1022D 可用 `identity.contains_any` 配多个可接受 SN

注意：

- 串口路径不是最终真值
- 当前连接事实来自 C# probes、snapshots 和 run artifact，不再依赖旧 `station verify`

### 2. calibration

文件：

- `configs/calibrations/main.json`

作用：

- 定义 `target_b_nt -> delta_current_a`
- runtime 在 point 里实际使用：
  - `locked_zero_offset_current_a + calibration.delta_current_a(target_b_nt)`

### 3. plan

文件：

- `configs/plans/minimal_3point_runtime.json`
- `configs/plans/x_axis_1d_bounce_15min.json`
- `configs/plans/grid_2d_raster_small.json`
- `configs/plans/grid_3d_example.json`
- `configs/plans/mag_zero_lock_verify.json`

作用：

- 定义 run_id、point settle、failure policy、零偏锁定策略、quality 阈值
- 定义显式 points 或高层 `cartesian_grid`

### 4. smb-profile

文件：

- `configs/profiles/smb100a_run_pll_default.json`
- `configs/profiles/smb100a_run_short_sweep_15min.json`

作用：

- 定义 SMB100A 固定 FM/LF 配置
- 定义默认 sweep 参数

### 5. oe-profile

文件：

- `configs/profiles/oe1022d_run_ch_b_observed.json`

作用：

- 定义 OE1022D Channel-B 固定配置
- 定义定长 `RALL?` 读取参数、committed 落盘和 ring buffer 观察容量

### 6. laser-profile

文件：

- `configs/profiles/cni_laser_run_on_background.json`

作用：

- 定义 Laser 在 run 级的固定背景策略
- 当前第一版只支持：
  - `on_background`
  - `off_background`

## 三、怎么改设备参数

### 1. 改 SMB100A 参数

改 `configs/profiles/smb100a_run_*.json`：

- 固定参数在 `fixed`
- sweep 默认参数在 `default_sweep`

point 级允许变化的只有：

- `start_hz`
- `stop_hz`
- `step_hz`
- `dwell_ms`
- `power_dbm`
- 以及白名单内的 `smb_override`

### 2. 改 OE1022D 参数

改 `configs/profiles/oe1022d_run_ch_b_observed.json`：

- `fixed` 下的是 run 固定配置
- `collector` 下的是定长 `RALL?` 采集、committed 落盘和 ring buffer 观察参数
- 当前 Channel-B 默认 profile 对齐官方 OE1022D 软件有效状态：`input_coupling=0`（AC coupling）、`time_constant_index=7`（30 ms）。
- `collector.rall_post_write_delay_ms` 对应原厂 LabVIEW 写 `RALL?` 后等待读取的时间，当前默认 `30ms`
- 定长 `RALL?` 热路径固定为：写 `RALL?` -> 等 `30ms` -> blocking exact read `12288B` -> 立刻下一轮
- 不再暴露 first-byte deadline、frame deadline、zero-byte retry 等旧 serial workaround 旋钮

第一版不允许 point 级修改 OE 配置。

### 3. 改 Laser 参数

改 `configs/profiles/cni_laser_run_on_background.json`：

- `mode`
- `power_mw`
- `settle_ms`

第一版不允许 point 级修改 Laser。

### 4. 改磁场零偏和验证参数

改 plan 里的 `mag_baseline_policy`：

- `baseline_current_a`
- `settle_ms`
- `readback_samples`
- `settle_tolerance_a`
- `voltage_v`
- `voltage_protection_v`
- `output_enabled`

## 四、1D / 2D / 3D 网格怎么写

### 1. 1D 往返

参考：

- `configs/plans/x_axis_1d_bounce_15min.json`

核心字段：

```json
"point_source": {
  "kind": "cartesian_grid",
  "axes_nt": {
    "x": [0.0, 10.0, 20.0, 30.0, 40.0],
    "y": [0.0],
    "z": [0.0]
  },
  "order": ["x", "y", "z"],
  "cycle_mode": "bounce_1d_x",
  "stop_condition": {
    "kind": "fixed_total_points",
    "total_points": 104
  }
}
```

当前硬规则：

- 第一版磁场电源只支持非负目标电流
- 如果 `locked_zero_offset_current_a + delta_current_a` 算出负值，C# `run-execute` 会直接报错
- 所以第一版示例网格全部改成非负 `target_b_nt`

### 2. 2D raster

参考：

- `configs/plans/grid_2d_raster_small.json`

规则：

- `x` 最快
- 再 `y`
- `z` 固定单值

### 3. 3D raster

参考：

- `configs/plans/grid_3d_example.json`
- `configs/plans/grid_3d_raster_0_50_100ut_validation.json`

规则：

- 第一版只支持显式数组
- 不支持 `start/stop/step` 范围语法
- 不支持更复杂的 serpentine
- `grid_3d_example.json` 只是展开样例
- `grid_3d_raster_0_50_100ut_validation.json` 才是当前已真机跑通的 `3D 0/50/100 uT` 单轮 27 点验证计划

## 五、零偏锁定现在是怎么做的

当前的“零场锁定”不是物理磁场已经测到 0，而是旧系统兼容的**零偏电流锁定**。

流程是：

1. 给三台 M8812 设置电压和过压保护
2. 下发每轴 `baseline_current_a`
3. 打开输出
4. 等待 `settle_ms`
5. 连续读 `MEAS:CURR?`
6. 用这些读数求出 `locked_zero_offset_current_a`
7. 后续 point 电流按：
   - `locked_zero_offset_current_a + calibration.delta_current_a(target_b_nt)`

对应产物：

- `baseline_snapshot.json`

关键字段：

- `mode = "legacy_zero_offset_lock"`
- `zero_offset_setpoint_a`
- `zero_offset_measured_samples_a`
- `locked_zero_offset_current_a`

## 六、当前 artifact 怎么看

### 1. 最终事实层

- `raw/oe1022d.rall`
- `raw/oe1022d.frames.idx.jsonl`
- `segments.jsonl`

默认轻量模式下，不再强制写逐帧 JSON sidecar。  
只有显式 `--artifact-mode debug` 时，才会额外写：

- `raw/oe1022d.frames.parsed.jsonl`

如果后续你要离线把某个 point 的 raw 单独裁出来，应按：

- `segments.jsonl`
- `raw_offset_start`
- `raw_offset_end`

来切，不要在采集线程里额外写 per-point raw 文件。

### 2. point 原标签层

- `points.jsonl`
- `point_fields.jsonl`
- `point_fields/*.npz`
- `point_fields/*.manifest.json`
- `quality.jsonl`

其中：

- `points.jsonl` 负责实验上下文
- `point_fields.jsonl` 负责轻量 metadata、摘要统计和 sidecar 路径
- `point_fields/*.npz` 负责完整 20 字段数组和必要状态数组
- `point_fields/*.manifest.json` 负责 point 级 SMB/Mag/Laser/OE 背景
- `quality.jsonl` 负责这段 point 数据是否达标
- `raw/oe1022d.frames.parsed.jsonl` 只在 debug 模式下作为逐帧结构化 companion truth

要注意：

- `point_fields.jsonl` 不再内联大数组
- `NPZ` sidecar 才是 point 级完整字段数据
- `manifest.json` sidecar 才是 point 级设备背景快照
- `raw/oe1022d.frames.parsed.jsonl` 不是默认产物

### 3. 运行摘要

- `summary.json`
- `events.jsonl`
- `run_manifest.json`

## 七、运行中的 stdout 代表什么

C# `run-execute` 现在会在 stdout 打印：

- run 打开
- station resolve 完成
- SMB/OE/Laser profile 下发完成
- 零偏锁定完成
- collector 启动
- 当前 point 序号、`target_b_nt`
- 当前 point `frames_total`
- 当前 point `quality_status`
- collector 累计 frame 数、timeout 数、估算点速率
- cleanup 开始 / 完成

这就是第一版的轻量监控界面，不是 GUI，不是 TUI。
