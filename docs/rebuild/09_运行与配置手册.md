# 运行与配置手册

本文只描述当前 `odmr-rebuild-01` 已实现的 Windows C# 主链和对应 PySide6 外壳。当前已经支持两种锁相放大器：

- `oe1022d`
- `oe1300`

两者共用同一套 `run-resolve / run-execute / resume-run / artifact-check / audit-continuity` 入口；差异只体现在 `station`、`oe_profile.model` 和 collector 合同。

## 一、命令入口

### 1. build

```powershell
dotnet build tools/win-csharp/Odmr.Win.sln -c Release
```

### 2. run-resolve

`run-resolve` 不碰设备，只检查 6 个 JSON 是否能组合成一次有效 run。

`OE1022D` 示例：

```powershell
dotnet run --project tools/win-csharp/Odmr.WinProbe -- run-resolve `
  --station configs/stations/lab_a.json `
  --calibration configs/calibrations/main.json `
  --plan configs/plans/minimal_3point_runtime.json `
  --smb-profile configs/profiles/smb100a_run_monitor_2830_2890_-10dbm.json `
  --oe-profile configs/profiles/oe1022d_run_ch_b_observed.json `
  --laser-profile configs/profiles/cni_laser_run_off_background.json
```

`OE1300` 示例：

```powershell
dotnet run --project tools/win-csharp/Odmr.WinProbe -- run-resolve `
  --station configs/stations/lab_a_oe1300_tcp.json `
  --calibration configs/calibrations/main.json `
  --plan configs/plans/minimal_3point_runtime.json `
  --smb-profile configs/profiles/smb100a_run_monitor_2830_2890_-10dbm.json `
  --oe-profile configs/profiles/oe1300_run_tcp_default.json `
  --laser-profile configs/profiles/cni_laser_run_off_background.json
```

### 3. run-execute

`run-execute` 是统一正式入口，不再把 `oe1300` demo 当正式主链。

`OE1022D` 示例：

```powershell
dotnet run --project tools/win-csharp/Odmr.WinProbe -- run-execute `
  --station configs/stations/lab_a.json `
  --calibration configs/calibrations/main.json `
  --plan configs/plans/x_axis_1d_bounce_15min.json `
  --smb-profile configs/profiles/smb100a_run_monitor_2830_2890_-10dbm.json `
  --oe-profile configs/profiles/oe1022d_run_ch_b_observed.json `
  --laser-profile configs/profiles/cni_laser_run_on_background.json `
  --out-dir runs/oe1022d_15min_demo
```

`OE1300` 示例：

```powershell
dotnet run --project tools/win-csharp/Odmr.WinProbe -- run-execute `
  --station configs/stations/lab_a_oe1300_tcp.json `
  --calibration configs/calibrations/main.json `
  --plan configs/plans/x_axis_1d_bounce_15min.json `
  --smb-profile configs/profiles/smb100a_run_monitor_2830_2890_-10dbm.json `
  --oe-profile configs/profiles/oe1300_run_tcp_default.json `
  --laser-profile configs/profiles/cni_laser_run_off_background.json `
  --out-dir runs/oe1300_15min_demo
```

### 4. resume-run

恢复入口保持统一，只接受当前 direct-decode 合同 run：

```powershell
dotnet run --project tools/win-csharp/Odmr.WinProbe -- resume-run `
  --previous-run runs/oe1022d_15min_demo `
  --out-dir runs/oe1022d_15min_demo__resume_01
```

恢复规则：

- 允许：`paused`、`failed`、`completed_with_failed_points`、`process exited with partial facts`
- 不允许：`completed`、`aborted`

### 5. artifact-check / audit-continuity / live-replay

```powershell
dotnet run --project tools/win-csharp/Odmr.WinProbe -- artifact-check --run runs/oe1022d_15min_demo
dotnet run --project tools/win-csharp/Odmr.WinProbe -- audit-continuity --run runs/oe1022d_15min_demo --out runs/oe1022d_15min_demo/continuity_audit.json
dotnet run --project tools/win-csharp/Odmr.WinProbe -- live-replay --run runs/oe1022d_15min_demo
```

三者都按 `lockin_model` 自动分支。

## 二、六个输入 JSON

一次 run 固定只有 6 个输入：

- `station`
- `calibration`
- `plan`
- `smb-profile`
- `oe-profile`
- `laser-profile`

当前不增加第二个 `lockin_profile` 参数，也不允许在一个 run 里同时激活两台锁相。

## 三、station 怎么分

当前按 active lock-in 分站点，不做单 station 内双锁相仲裁。

### 1. OE1022D 站点

- `configs/stations/lab_a.json`

其中锁相设备 `kind` 必须是 `oe1022d`，设备 ID 约定为 `oe1022d_main`。

### 2. OE1300 站点

- `configs/stations/lab_a_oe1300_tcp.json`

其中锁相设备 `kind` 必须是 `oe1300`，设备 ID 约定为 `oe1300_main`，通过 TCP 连接。

### 3. 硬规则

- `station` 是 active lock-in 的唯一选择入口
- 不允许让 `oe_profile` 自己去反查设备 ID
- `station` 与 `oe_profile.model` 不一致时，`run-resolve` 和 PySide6 本地校验都必须失败

## 四、oe_profile 怎么写

`--oe-profile` 参数名保持不变，但 profile 顶层必须带 `model`。

允许值只有：

- `oe1022d`
- `oe1300`

### 1. OE1022D profile

示例：

- `configs/profiles/oe1022d_run_ch_b_observed.json`
- `configs/profiles/oe1022d_run_ch_b_tc100ms.json`

结构：

- `model = "oe1022d"`
- `fixed`
- `collector`

collector 当前是冻结合同，只允许已经验证过的值：

- `frame_bytes = 12288`
- `samples_per_frame = 50`
- `rall_post_write_delay_ms = 30`

热路径固定为：

```text
write RALL?
sleep 30ms
exact read 12288B
direct-decode
append collector_frames + parameter_values + sample_values
```

### 2. OE1300 profile

示例：

- `configs/profiles/oe1300_run_tcp_default.json`

结构：

- `model = "oe1300"`
- `fixed`
- `collector`

collector 当前是冻结合同，只允许已经验证过的值：

- `tcp_expected_bytes = 32768`
- `tcp_payload_bytes = 29600`
- `parameter_count = 37`
- `samples_per_parameter = 100`
- `rall_post_write_delay_ms = 5`

热路径固定为：

```text
write RALL?\r
sleep 5ms
read until 32768B
detect unique block
decode 37 x 100 big-endian double for unique blocks only
append collector_blocks + unique-only parameter_values + unique-only sample_values
```

## 五、其他配置文件

### 1. calibration

- `configs/calibrations/main.json`

作用：

- 把 `target_b_nt` 转成 `delta_current_a`
- point 里真正下发的是：
  - `locked_zero_offset_current_a + delta_current_a(target_b_nt)`

### 2. plan

常用示例：

- `configs/plans/minimal_3point_runtime.json`
- `configs/plans/x_axis_1d_bounce_15min.json`
- `configs/plans/grid_2d_raster_small.json`

作用：

- 定义 `run_id`
- 定义 points 或 `cartesian_grid`
- 定义 point settle / quality / baseline policy

### 3. smb-profile

常用示例：

- `configs/profiles/smb100a_run_monitor_2830_2890_-10dbm.json`
- `configs/profiles/smb100a_run_3d_validation_fast.json`

作用：

- 固定 FM/LF 背景
- 定义默认 sweep

### 4. laser-profile

常用示例：

- `configs/profiles/cni_laser_run_off_background.json`
- `configs/profiles/cni_laser_run_on_background.json`

作用：

- 定义 run 级 laser 背景策略

## 六、当前 artifact 口径

正式事实层已经统一切到 decoded truth。

### 1. OE1022D

- `collector_frames.jsonl`
- `parameter_values.csv`
- `sample_values.csv`
- `segments.jsonl`

### 2. OE1300

- `collector_blocks.jsonl`
- `parameter_values.csv`
- `sample_values.csv`
- `segments.jsonl`

其中 `collector_blocks.jsonl` 保留所有 query 到的块；`parameter_values.csv` 与 `sample_values.csv` 只写 `unique_block = true` 的去重后有效样本。

共同外围文件：

- `points.jsonl`
- `quality.jsonl`
- `device_state.jsonl`
- `events.jsonl`
- `run_manifest.json`
- `summary.json`

不再要求：

- `raw/oe1022d.rall`
- `raw/oe1022d.frames.idx.jsonl`
- raw replay 伴生产物

## 七、pause / resume / stop

### 1. 普通停止

PySide6 或脚本写入：

- `control/stop.request`

runtime 语义是：

- stop-after-current-point
- 在 point 边界停下
- 终态写成 `paused`
- terminal event 写成 `run_paused`

### 2. 急停

PySide6 或脚本写入：

- `control/emergency_stop.request`

runtime 尽快中断当前 point，并执行：

- SMB RF OFF
- Laser OFF
- M8812 cleanup
- collector stop

终态写成 `aborted`。

### 3. 恢复

恢复不覆盖旧 run，而是在新目录产出一个恢复 slice。恢复判定只依赖：

- `events.jsonl`
- `points.jsonl`
- `quality.jsonl`
- `summary.json`
- `run_manifest.json`

不读取任何历史 raw 文件。

## 八、验收口径

### 1. OE1022D

每轮 run 至少要求：

- `artifact-check passed`
- `audit-continuity verdict = continuous`
- `timeout_count = 0`
- `raw_len_bad_count = 0`
- `delta_gt1_count = 0`

### 2. OE1300

每轮 run 至少要求：

- `artifact-check passed`
- `audit-continuity verdict = continuous`
- `timeout_count = 0`
- `raw_len_bad_count = 0`
- `decode_failures = 0`
- `effective_sample_hz_per_parameter >= 900`

`OE1300` 在没有接入实际光路时，可以接受“没有可解释谱线”，但不能接受：

- 结构错位
- 解码崩溃
- artifact 不自洽

## 九、当前 PySide6 入口

```powershell
python tools/odmr-console-python/odmr_console_qt.py
```

当前 UI 负责：

- 选择六个 JSON
- 生成 plan / profile
- 本地校验 station / oe_profile 型号一致性
- 调用 `run-resolve`
- 调用 `run-execute`
- 调用 `resume-run`
- tail `progress.jsonl`
- 触发 stop / emergency-stop
- 调用 `artifact-check` / `audit-continuity`

它不直接连接任何设备。

## 十、当前不做的事

本阶段明确不做：

- 不做 raw-truth 兼容层
- 不做双写
- 不做一个 station 内双锁相切换
- 不做 reader/parser/writer 线程拆分
- 不做自动并行 run 调度
- 不把 `oe1300` demo 当正式入口
