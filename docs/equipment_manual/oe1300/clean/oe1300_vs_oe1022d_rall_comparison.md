# OE1022D vs OE1300 `RALL?` 差异分析（文档阶段，不改代码）

> 本文件仅对比两台锁相放大器 `RALL?` 指令的返回格式、字段语义和采集含义，为后续是否接入 OE1300 提供决策依据。

---

## 一、指令名称相同，语义不同

| 项目 | OE1022D | OE1300 |
|------|---------|--------|
| 指令字符串 | `RALL?\r` | `RALL?\r` |
| 接口限制 | **仅 USB2.0 可用**，RS232 无此指令 | 未说明限制，UART/网口均可 |
| 返回内容 | 所有测量数据 + 当前配置 + 空字符 | 所有解调结果数据 + Aux IN 数据 |
| 返回格式 | **12288 字节二进制帧** | **ASCII 浮点 CSV 字符串** |

---

## 二、返回格式对比

### OE1022D：固定长度二进制帧

| 属性 | 说明 |
|------|------|
| 总长度 | 12288 Bytes |
| 测量数据区 | 前 8000 Bytes |
| 配置信息区 | 1216 Bytes |
| 空字符区 | 3072 Bytes |
| 数据刷新周期 | 每 50 ms 更新一次 |
| 单次返回点数 | 每个测量字段 50 个历史点 |
| 采样间隔 | 1 ms |
| 浮点数类型 | 64 位浮点数（f64），实测大端 |
| 无分隔符 | 数据中间没有逗号，按固定偏移排列 |

**测量字段顺序（前 8000 Bytes）**：

1. A 通道 X（50 × f64）
2. A 通道 Y（50 × f64）
3. A 通道 Frequency（50 × f64）
4. A 通道 Noise（50 × f64）
5. A 通道 Xh1（50 × f64）
6. A 通道 Yh1（50 × f64）
7. A 通道 Xh2（50 × f64）
8. A 通道 Yh2（50 × f64）
9. B 通道 X（50 × f64）
10. B 通道 Y（50 × f64）
11. B 通道 Frequency（50 × f64）
12. B 通道 Noise（50 × f64）
13. B 通道 Xh1（50 × f64）
14. B 通道 Yh1（50 × f64）
15. B 通道 Xh2（50 × f64）
16. B 通道 Yh2（50 × f64）
17. AUXADC1（50 × f64）
18. AUXADC2（50 × f64）
19. AUXADC3（50 × f64）
20. AUXADC4（50 × f64）

**配置信息区（8000–9215 Bytes）**：
- A/B 通道 Ref Phase、Ref Source、频率、谐波、扫频、Sineout、Equation、Gain/TC/Input Filter、Sample、状态等。

**关键连续性字段**：
- 当前项目使用 `payload[12287]` 作为 device packet counter。
- 手册未明确说明该字节含义，但实测可用作帧连续性审计。

---

### OE1300：变长 ASCII CSV 字符串

| 属性 | 说明 |
|------|------|
| 总长度 | **不固定**，取决于返回字段数量 |
| 分隔符 | 逗号 `,` |
| 数值格式 | ASCII 浮点数 |
| 返回字段 | 37 个参数，每个参数 1 个值 |
| 更新周期 | 手册未明确 |
| 单次返回点数 | **每个参数只返回 1 个当前值**（不是 50 个历史点） |
| 终止符 | 手册未明确，推测为 `\r` 或 `\r\n` |

**字段顺序（0–36）**：

| 编码 | 参数 | 编码 | 参数 |
|---:|---|---:|---|
| 0 | X | 19 | θD4 |
| 1 | Y | 20 | XD5 |
| 2 | R | 21 | YD5 |
| 3 | θ | 22 | RD5 |
| 4 | XD1 | 23 | θD5 |
| 5 | YD1 | 24 | XD6 |
| 6 | RD1 | 25 | YD6 |
| 7 | θD1 | 26 | RD6 |
| 8 | XD2 | 27 | θD6 |
| 9 | YD2 | 28 | XD7 |
| 10 | RD2 | 29 | YD7 |
| 11 | θD2 | 30 | RD7 |
| 12 | XD3 | 31 | θD7 |
| 13 | YD3 | 32 | X-Noise |
| 14 | RD3 | 33 | Y-Noise |
| 15 | θD3 | 34 | Frequency |
| 16 | XD4 | 35 | AUX-IN1 |
| 17 | YD4 | 36 | AUX-IN2 |
| 18 | RD4 | | |

**关键差异**：
- 没有 50 个历史点，只有当前快照。
- 没有配置信息区。
- 没有 packet counter。
- 返回格式是可直接阅读的 ASCII，不是二进制。

## 三、采集循环差异

| 项目 | OE1022D | OE1300 |
|------|---------|--------|
| 发送指令 | `RALL?\r` | `RALL?\r` |
| 等待策略 | sleep 30 ms | 未知，需实测 |
| 读取策略 | blocking exact read 12288 Bytes | read-until-terminator |
| 帧大小校验 | `frames × 12288` | 无法按固定大小校验 |
| 连续性审计 | `payload[12287]` packet counter | 无 packet counter，需用时间戳/序号 |
| 字段解析 | 按字节偏移解析 f64 数组 | 按逗号 split 转 double |

---

## 四、对现有项目的具体影响

如果未来接入 OE1300，下列地方不能复用 OE1022D 逻辑：

| 模块 | 当前 OE1022D 逻辑 | OE1300 需要的新逻辑 |
|------|-------------------|---------------------|
| `Oe1022dVisa` 读取器 | exact 12288B read | line reader / terminator read |
| `ContinuityAudit` | 比较 `payload[12287]` | 比较时间戳或 frame index |
| `ArtifactCheck` | `raw_size == frames × 12288` | 按行/帧校验或仅检查存在性 |
| `extract_point_fields_from_rall.py` | 二进制偏移解析 | ASCII CSV 解析 |
| `point_fields` 产出 | 20 字段 × 50 点数组 | 37 字段 × 1 点 |
| 数据保存格式 | `.rall` 二进制文件 | `.rall` 文本文件或改扩展名 |

---

## 五、关键待确认问题（需真机验证）

在决定是否接入 OE1300 之前，必须确认：

1. OE1300 `RALL?` 实际返回字节数范围是多少？是否稳定？
2. 终止符是 `\r`、`\n` 还是 `\r\n`？
3. 连续两次 `RALL?` 之间推荐延时是多少？
4. 37 个字段是否总是全部返回，还是只返回使能的解调器？
5. 是否包含类似 OE1022D 的 packet counter？
6. ASCII 数值精度是多少？是否足够满足 ODMR 数据分析？
7. `RALL?` 在 UART 和网口上行为是否一致？

---

## 六、结论

两台设备的 `RALL?` **虽然指令名相同，但返回格式完全不同**：

- **OE1022D**：高密度二进制历史帧，适合高速连续采集和离线重分析。
- **OE1300**：稀疏 ASCII 当前快照，解析简单但丢失了 50 点历史缓冲。

这个差异是引入 OE1300 时**无法绕过**的核心矛盾。如果项目需要保留当前“每帧 50 点、连续性审计、二进制 raw 归档”的运行模式，OE1300 不能直接替代 OE1022D，而需要重新设计采集与归档语义。
