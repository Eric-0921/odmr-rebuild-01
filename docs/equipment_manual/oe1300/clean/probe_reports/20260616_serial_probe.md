# OE1300/OE1351 串口探测报告

> 探测时间：2026-06-16  
> 探测接口：Mac USB-RS232 → `/dev/cu.usbserial-B0027SH3`  
> 波特率：115200（默认）  
> 终止符：`\r`  
> 项目代码：**未改动**

---

## 1. 设备身份

```
*IDN? -> SSI LIA-OE1351,SN:L2092228,Version:V1.3230310
```

**注意**：设备型号为 **OE1351**，属于 OE1300 系列（手册标题为 OE1300 series）。

---

## 2. `RALL?` 实测结果

### 2.1 返回格式

- **长度**：约 522–524 字节
- **格式**：ASCII 浮点 CSV
- **字段数**：37 个值
- **终止符**：`\r`
- **示例**：

```
-6.8591155e-05,2.5672501e-02,2.5672593e-02,9.0153079e+01,
-4.3860876e-05,2.5672555e-02,2.5672593e-02,9.0097886e+01,
...
5.3515474e-07,5.5830351e-04,1.023181539e-06,6.2500000e-04,6.2500000e-04
```

### 2.2 字段解析

37 个字段与手册表一致：

| 编码 | 参数 | 实测值（示例） |
|---:|---|---|
| 0 | X | -6.859e-05 |
| 1 | Y | 2.567e-02 |
| 2 | R | 2.567e-02 |
| 3 | θ | 9.015e+01 |
| 4–31 | XD1–θD7 | … |
| 32 | X-Noise | 5.352e-07 |
| 33 | Y-Noise | 5.583e-04 |
| 34 | Frequency | 1.023e-06 |
| 35 | AUX-IN1 | 6.250e-04 |
| 36 | AUX-IN2 | 6.250e-04 |

### 2.3 关键发现

- **没有 50 点历史缓冲**：每个参数只返回 1 个当前值。
- **没有 packet counter**：帧内无连续性标识字节。
- **帧长度接近固定**：522–524 字节，波动来自数值精度（科学计数法位数变化）。

## 3. 响应时间（修正）

> ⚠️ 早期手工探测显示 ~1.5 s/命令，但后续使用动态读取（读到 `\r` 终止符即返回）的脚本复测发现：**实际命令往返时间约为 20–80 ms**。早期 1.5 s 是读取策略（等超时）造成的假象。

### 3.1 单条命令实测

| 指令 | 往返时间 | 返回字节数 |
|------|---------|-----------|
| `*IDN?` | ~26 ms | 46 |
| `FMOD?` | ~23 ms | 1 |
| `FREQ?` | ~26 ms | 16 |
| `PHAS?` | ~26 ms | 13 |
| `SNAP? 0` | ~23 ms | 15 |
| `RALL?` | ~74 ms | ~522 |
| `*RST` | ~3000 ms | 多行 boot 信息 |
| `NMOD?` / `NIPA?` / `NSMA?` / `NGWA?` | ~3000 ms | 超时无响应 |

### 3.2 连续 `RALL?` 爆发测试

100 次连续 `RALL?`（命令间约 1 ms 间隔）：

- 总时间：5.504 s
- 平均吞吐：**18.17 Hz**
- 单次延迟：min 45.16 / mean 53.59 / median 47.10 / max 62.71 ms

**结论**：OE1351 串口实际采集能力约为 **18 Hz**，远高于早期估计的 0.5–1 Hz。

## 4. 对项目的含义

### 4.1 与 OE1022D 对比

| 项目 | OE1022D | OE1351（OE1300 系列） |
|------|---------|----------------------|
| `RALL?` 周期 | 50 ms | ~1000 ms |
| 返回格式 | 12288B 二进制，50 点/字段 | 522B ASCII，1 点/字段 |
| 数据吞吐 | 高 | 低 |
| 连续性审计 | `payload[12287]` packet counter | 无 |

### 4.2 开发影响

- **串口采集频率上限约为 18 Hz**，接近 OE1022D 的 20 Hz，基本可满足多数 ODMR 实验需求。
- 网口路径仍值得测试，因为网口可能进一步降低延迟并支持示波器/FFT 功能（手册 3.16/3.17 节）。
- 若网口始终无法启用，**串口路径单独即可作为可行方案**。

## 5. 命令兼容性验证

使用 `tools/oe_rall_compare/oe1300_serial_command_probe.py` 对手册命令批量验证（结果保存于 `/tmp/oe1300_cmd_probe/oe1300_command_probe_results.jsonl`）：

### 5.1 正常响应的命令

`*IDN?`、`*RST`、`*PLL?`、`ISRC?`、`ICPL?`、`IRNG?`、`FMOD?`、`FREQ?`、`PHAS?`、`RSLP?`、`OFLT?`、`OFSL?`、`SYNC?`、`HARM?`、`BAUD?`、`OVLD?`、`SNAP?`、`RALL?`

### 5.2 超时/不支持的命令

`NMOD?`、`NIPA?`、`NSMA?`、`NGWA?`、`NMOD 0`、`NSMA ...`、`NGWA ...`、`IGND?`、`RMOD?`、`OUTP?`、`OEUT? 0`、`OEUT? 1`

> 例外：`NIPA 192.168.1.5` 返回 `"Setting Ethernet Mode Success!"`，但对应的 `NIPA?` 仍超时，且重启后配置不保留。

### 5.3 `*RST` 行为

`*RST` 不是简单的参数清零，而是触发 **Xilinx Zynq 全系统重启**，返回 boot loader / PMU 信息，耗时约 3 s。

### 5.4 网络接口状态

网络查询命令在当前固件 `V1.3230310` 上基本无响应，说明**网口配置命令未实现或存在 bug**。
