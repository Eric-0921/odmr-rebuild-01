# 选项 A 说明：改的是 OE 设备的网络参数

## 当前状态

- **Mac (en5)**：已有别名 `192.168.1.10/24` ✅
- **OE 设备**：网口未配置，IP 未知 ❌

## 选项 A 做什么？

通过**串口**向 OE 设备发送命令，**修改 OE 设备自身的网络参数**：

```text
NMOD 0                      // 设置 OE 设备为 TCP 模式
NIPA 192.168.1.5            // 设置 OE 设备 IP 为 192.168.1.5
NSMA 255.255.255.0          // 设置 OE 设备子网掩码
NGWA 192.168.1.1            // 设置 OE 设备网关
```

改完后：
- OE 设备 IP 变成 `192.168.1.5`
- Mac IP 仍然是 `192.168.1.10`
- Mac 可以 ping `192.168.1.5`

## 不改 Mac 什么

- Mac 的 `192.168.1.10` 别名保留
- Mac 的 Wi-Fi (`en0`) 完全不动
- Mac 的默认网关、DNS、路由都不动

## 风险提示

- 这会**永久修改 OE 设备的网络配置**（直到你再次修改）。
- 如果 OE 设备原来有网络配置，会被覆盖。
- 如果之后你把 OE 设备接到其他网络（比如公司内网是 192.168.1.x），可能会 IP 冲突。

## 对比选项 B

| | 选项 A（串口配置） | 选项 B（Console 软件） |
|---|-------------------|----------------------|
| 改谁 | 改 OE 设备 | 改 OE 设备 |
| 工具 | 串口命令 | PC Console GUI |
| 风险 | 命令直接写配置，无确认 | GUI 操作，相对直观 |
| 速度 | 快 | 需要安装 Console |

两个选项改的都是 OE 设备，不是 Mac。
