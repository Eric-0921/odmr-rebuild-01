# 将 Mac en5 设为静态 IP 的影响分析

## 当前网络状态

```
en0 (Wi-Fi):  192.168.199.184  -> 主网络/互联网
en5 (有线):   169.254.192.37   -> 直连 OE1300/OE1351
```

`en5` 和 `en0` 是两个独立的物理接口，互不影响。

---

## 影响范围

### 1. 不会影响的部分

- **Wi-Fi 上网**：en0 保持 DHCP，不受影响。
- **其他网络服务**：DNS、路由、VPN 等仍走 en0。
- **项目代码**：不改任何代码文件。

### 2. 会影响的部分

- **en5 这个接口本身**：从 link-local 地址改为 192.168.1.x 网段。
- **en5 上连接的其他设备**：如果 en5 除了 OE1300 还接了别的设备，那些设备可能无法通信（除非你设的静态 IP 正好在它们网段）。
- **OE1300 通信**：这是我们要测试的目标。

---

## 推荐做法：添加别名 IP（最安全）

不删除现有的 `169.254.192.37`，而是额外给 en5 加一个 `192.168.1.10/24` 的别名 IP。

```bash
sudo ifconfig en5 alias 192.168.1.10/24
```

这样：
- Mac 仍然保留 link-local 地址，不会断开现有连接。
- Mac 同时获得 192.168.1.10，可以与 192.168.1.1 通信。
- 测试完后可以随时删除：

```bash
sudo ifconfig en5 -alias 192.168.1.10
```

---

## 替代做法：完全改为静态 IP

如果你确定 en5 只接 OE1300：

```bash
sudo ifconfig en5 inet 192.168.1.10 netmask 255.255.255.0
```

这样：
- 会替换掉当前的 `169.254.192.37`。
- 测试完后需要重新启用 DHCP 或改回原来的 link-local。
- 风险稍高于添加别名。

---

## 风险总结

| 操作 | 风险 | 可恢复性 |
|------|------|---------|
| 添加别名 IP | 极低 | 随时删除别名 |
| 改为静态 IP | 低（仅影响 en5） | 可手动改回 DHCP |
| 错误配置网关 | 可能导致部分路由走 en5 而非 WiFi | 需删除错误路由 |

**关键提醒**：只改 IP 地址，**不要设置默认网关**为 192.168.1.1，否则 Mac 可能会尝试通过 OE1300 上网，导致网络异常。

---

## 测试后验证

设置别名后，应能 ping 通 OE1300：

```bash
ping 192.168.1.1
```

然后尝试 TCP 连接（端口号需实测，常见 5025 / 10001 / 23）：

```bash
nc -zv 192.168.1.1 5025
nc -zv 192.168.1.1 10001
nc -zv 192.168.1.1 23
```
