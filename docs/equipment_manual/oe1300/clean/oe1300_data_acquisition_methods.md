# OE1300 数据采集方式梳理（基于校对后手册）

> 本文件汇总 OE1300 手册中提到的所有数据获取/采集方式，区分“远程命令可用”与“PC Console 专用”。

---

## 一、远程命令可用的数据读取方式

OE1300 第 5 章《远程编程》中，明确可用于读取数据的指令如下：

### 1. `RALL?` —— 读取所有解调结果与 Aux IN

| 属性 | 说明 |
|------|------|
| 返回格式 | ASCII 浮点 CSV 字符串 |
| 返回字段数 | 37 个参数 |
| 单次点数 | 每个参数 1 个当前值 |
| 适用接口 | UART / 网口 |
| 用途 | 一次性读取当前所有解调器输出和 AUX 输入 |

字段顺序：0=X, 1=Y, 2=R, 3=θ, 4=XD1, 5=YD1, 6=RD1, 7=θD1, …, 31=θD7, 32=X-Noise, 33=Y-Noise, 34=Frequency, 35=AUX-IN1, 36=AUX-IN2。

---

### 2. `SNAP? i,j{,k,l,m,n,o,p,q,r}` —— 同时快照最多 10 个参数

| 属性 | 说明 |
|------|------|
| 最少参数 | 2 个 |
| 最多参数 | 10 个 |
| 返回格式 | ASCII 浮点 CSV 字符串 |
| 特点 | 同一时刻读取，避免 `OUTP?` 多次查询的延时 |
| 参数编码 | 与 `RALL?` / `OUTP?` 一致（0–36） |

示例：`SNAP?0,1,34,3;` 返回 X, Y, Frequency, θ。

---

### 3. `OUTP? i` —— 读取单个参数

| 属性 | 说明 |
|------|------|
| 返回格式 | 单个 ASCII 浮点数 |
| 参数 i | 0–36，与 `RALL?` 字段编码一致 |
| 用途 | 只需单个值时 |
| 缺点 | 连续读多个参数存在时间差 |

---

### 4. `OAUX? i` —— 读取 AUX-IN 输入电压

| 属性 | 说明 |
|------|------|
| 参数 i | 0=AUX-IN1, 1=AUX-IN2 |
| 返回 | 电压值（V） |
| 用途 | 单独读取辅助模拟输入 |

---

### 5. 状态查询指令

| 指令 | 返回 | 用途 |
|------|------|------|
| `OVLD?` | 0 或 1 | ADC 是否过载 |
| `*PLL?` | 0 或 1 | 锁相环是否锁定 |
| `*IDN?` | IDN 字符串 | 设备身份识别 |

这些不是“数据采集”，但属于运行状态监控。

---

## 二、PC Console 软件专用的数据功能

以下功能在手册第 6 章《PC 软件安装使用说明》中描述，**未在第 5 章远程命令中出现**。也就是说，目前没有远程命令可以直接获取这些数据。

### 1. Sample 数据保存（CSV 文件）

| 属性 | 说明 |
|------|------|
| 入口 | Console 软件界面 |
| 保存内容 | R、X、Y、θ、频率、噪声；7 路谐波的 R/X/Y/θ；2 路 AUX IN |
| 保存格式 | CSV |
| 采样间隔 | 1 ms – 100 s，默认 100 ms |
| 控制方式 | 点击 `Save Data` / `Saving` 按钮 |
| 远程替代 | 无对应远程命令 |

**开发提示**：若需要类似 OE1022D 的连续数据缓存功能，OE1300 只能在上位机软件中通过 `RALL?` 轮询实现，设备本身没有暴露 `TRCA?` 之类的缓存读取指令。

---

### 2. 示波器功能（Oscilloscope）

| 属性 | 说明 |
|------|------|
| 接口要求 | 仅 TCP/IP 连接可用 |
| 模拟带宽 | 1 MHz |
| 垂直分辨率 | 24 bits |
| 最高采样率 | 4 MSPS |
| 显示长度 | 8192 / 4096 点 |
| 远程命令 | 无 |

---

### 3. FFT 频谱分析功能

| 属性 | 说明 |
|------|------|
| 接口要求 | 仅 TCP/IP 连接可用 |
| 分析范围 | DC – 2 MHz |
| 远程命令 | 无 |

---

## 三、与 OE1022D 的对比

| 采集方式 | OE1022D | OE1300 |
|----------|---------|--------|
| 全局数据 | `RALL?` → 12288B 二进制帧 | `RALL?` → ASCII CSV |
| 快照 | `SNAPD?` | `SNAP?` |
| 单参数 | `OUTPD?` | `OUTP?` |
| AUX 查询 | `OAUXD?` | `OAUX?` |
| 过载 | `INOVD?` / `GNOVD?` | `OVLD?` |
| PLL | `*PLLD?` | `*PLL?` |
| 数据缓存/采样 | `SRATD`、`SLEND`、`SSLED`、`TRCAD?` 等 | **无远程命令**，仅 Console 软件 CSV 保存 |
| 示波器/FFT | 面板功能 | Console 软件功能，TCP/IP 专用 |

---

## 四、对开发的关键结论

1. **OE1300 没有 OE1022D 那样的数据缓存区远程读取机制**。
   - OE1022D 可以通过 `SRATD`/`SLEND`/`TRCAD?` 配置并读取内部采样缓冲区。
   - OE1300 的“Sample 数据保存”是 PC 软件行为，不是设备命令。

2. **OE1300 的连续采集唯一可行路径是轮询 `RALL?`**。
   - 每次返回 37 个当前值。
   - 需要自己上位机实现采样率和 CSV 保存。

3. **若需要历史波形/示波器/FFT 数据，必须走 Console 软件或二次开发协议**。
   - 手册未提供这些功能的远程命令。
   - 若有隐藏协议，需要抓包或向厂商确认。

4. **从 OE1022D 迁移到 OE1300 时，不能简单替换 `RALL?` 解析器**。
   - 还要放弃“设备内部缓存 + 批量读取”的采集模式，改为上位机轮询模式。
